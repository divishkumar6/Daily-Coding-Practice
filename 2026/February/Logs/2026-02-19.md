### 1. Check if Array Is Sorted and Rotated

- **Difficulty**: Easy
- **Topic**: Arrays
- **Source**: LeetCode #1752
- **Solution**: `Week-1/Arrays/CheckIfArrayIsSortedAndRotated.java`
- **LeetCode Post**: [Java O(n) Solution Using Pivot](https://leetcode.com/problems/check-if-array-is-sorted-and-rotated/solutions/7590658/java-on-solution-using-pivot-minimum-ele-awz0/)
- **Approach**: Find pivot (minimum element), rotate array from that point, check if sorted
- **Time Complexity**: O(n) - single pass to find pivot + single pass to rotate + single pass to verify
- **Space Complexity**: O(n) - temporary array for rotation

**Notes:**
- Found the minimum value position as the pivot point
- Rotated the array starting from the pivot
- Checked if the rotated array is sorted in non-decreasing order
- Handles duplicates by continuing when adjacent elements are equal
- Returns true if array can be obtained by rotating a sorted array

**Key Insights:**
- The pivot point is where the rotation happened
- After un-rotating (starting from pivot), array should be sorted
- Edge case: Array with no rotation (already sorted) returns true
