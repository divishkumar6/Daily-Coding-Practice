### 1. Max Consecutive Ones

- **Difficulty**: Easy
- **Topic**: Arrays
- **Source**: LeetCode #485
- **Solution**: `Week-1/Arrays/MaxConsecutiveOnes.java`
- **Approach**: Single pass with counter tracking consecutive 1s
- **Time Complexity**: O(n) - single pass through array
- **Space Complexity**: O(1) - only two variables used

**Notes:**
- Use a counter to track current consecutive 1s
- Use a max variable to store the maximum count seen so far
- When encountering a 0, update max if current count is greater
- Reset counter to 0 after updating max
- Important: Check max one final time after loop (handles case where array ends with 1s)

**Algorithm Breakdown:**
1. Initialize count = 0 and max = 0
2. For each element in array:
   - If element is 1: increment count
   - If element is 0: update max if needed, reset count
3. After loop: update max one final time (for trailing 1s)
4. Return max

**Edge Cases Handled:**
- ✅ Array ending with consecutive 1s (final check after loop)
- ✅ All 1s (final check handles this)
- ✅ All 0s (max remains 0)
- ✅ Single element array
- ✅ Alternating 1s and 0s

**Solution Analysis:**
- ✅ Optimal time complexity O(n)
- ✅ Optimal space complexity O(1)
- ✅ Single pass solution
- ✅ Handles all edge cases correctly
- ✅ Clean and readable code

**Key Insight:**
- The final check `if(max <= count)` after the loop is crucial
- Without it, we'd miss the case where the array ends with consecutive 1s
- Example: [1,1,1] would return 0 without the final check
