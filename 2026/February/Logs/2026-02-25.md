### 1. Max Consecutive Ones

- **Difficulty**: Easy
- **Topic**: Arrays
- **Source**: LeetCode #485
- **Solution**: `Week-1/Arrays/MaxConsecutiveOnes.java`
- **Approach**: Single pass with counter tracking consecutive 1s
- **Time Complexity**: O(n) - single pass through array
- **Space Complexity**: O(1) - only two variables used

**Notes:**
- Use a counter to track current consecutive 1s
- Use a max variable to store the maximum count seen so far
- When encountering a 0, update max if current count is greater
- Reset counter to 0 after updating max
- Important: Check max one final time after loop (handles case where array ends with 1s)

**Algorithm Breakdown:**
1. Initialize count = 0 and max = 0
2. For each element in array:
   - If element is 1: increment count
   - If element is 0: update max if needed, reset count
3. After loop: update max one final time (for trailing 1s)
4. Return max

**Edge Cases Handled:**
- ✅ Array ending with consecutive 1s (final check after loop)
- ✅ All 1s (final check handles this)
- ✅ All 0s (max remains 0)
- ✅ Single element array
- ✅ Alternating 1s and 0s

**Solution Analysis:**
- ✅ Optimal time complexity O(n)
- ✅ Optimal space complexity O(1)
- ✅ Single pass solution
- ✅ Handles all edge cases correctly
- ✅ Clean and readable code

**Key Insight:**
- The final check `if(max <= count)` after the loop is crucial
- Without it, we'd miss the case where the array ends with consecutive 1s
- Example: [1,1,1] would return 0 without the final check

### 2. Union of Two Sorted Arrays

- **Difficulty**: Easy
- **Topic**: Arrays, Merging
- **Source**: TUF+ (Take U Forward)
- **Solution**: `Week-1/Arrays/UnionOfTwoSortedArrays.java`
- **Approach**: Merge arrays, sort, remove duplicates
- **Time Complexity**: O((m+n)log(m+n)) - due to sorting
- **Space Complexity**: O(m+n) - result array

**Notes:**
- Merge both arrays into single result array
- Sort the merged array to get elements in order
- Use two-pointer technique to remove duplicates
- Return only unique elements using Arrays.copyOf

**Algorithm Breakdown:**
1. Create result array of size m+n
2. Copy all elements from both arrays
3. Sort the result array
4. Remove duplicates using two pointers (k and l)
5. Return trimmed array with only unique elements

**Current Solution Analysis:**
- ✅ Works correctly for all test cases
- ✅ Handles duplicates within and across arrays
- ✅ Returns sorted unique elements
- ⚠️ Not optimal - doesn't leverage that input arrays are already sorted
- ⚠️ O((m+n)log(m+n)) time due to sorting

**Future Optimization Ideas:**

**Optimal Approach: Two-Pointer Merge (O(m+n))**
```java
// Since both arrays are sorted, use two pointers
int i = 0, j = 0, k = 0;
int[] result = new int[nums1.length + nums2.length];

while (i < nums1.length && j < nums2.length) {
    if (nums1[i] < nums2[j]) {
        if (k == 0 || result[k-1] != nums1[i]) {
            result[k++] = nums1[i];
        }
        i++;
    } else if (nums1[i] > nums2[j]) {
        if (k == 0 || result[k-1] != nums2[j]) {
            result[k++] = nums2[j];
        }
        j++;
    } else { // Equal elements
        if (k == 0 || result[k-1] != nums1[i]) {
            result[k++] = nums1[i];
        }
        i++;
        j++;
    }
}
// Add remaining elements from nums1 or nums2
```
- Time: O(m+n), Space: O(m+n)
- Leverages sorted property
- No sorting needed!
