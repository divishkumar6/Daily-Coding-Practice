### 1. Move Zeroes

- **Difficulty**: Easy
- **Topic**: Arrays
- **Source**: LeetCode #283
- **Solution**: `Week-1/Arrays/MoveZeroes.java`
- **Approach**: Use temporary array to collect non-zero elements, then copy back
- **Time Complexity**: O(n) - two passes through the array
- **Space Complexity**: O(n) - temporary array of same size

**Notes:**
- First pass: Copy all non-zero elements to temp array in order
- Second pass: Copy temp array back to original (zeros automatically at end)
- Maintains relative order of non-zero elements
- Simple and straightforward approach
- Works correctly but uses extra space

**Current Solution Analysis:**
- ✅ Works correctly and passes all test cases
- ✅ Maintains relative order of non-zero elements
- ✅ Easy to understand and implement
- ⚠️ Uses O(n) extra space for temporary array
- ⚠️ Not truly in-place as problem suggests

**Future Optimization Ideas:**

**Approach 1: Two-Pointer Technique (O(1) space) - OPTIMAL**
```java
// Use two pointers: one for reading, one for writing non-zeros
int writePos = 0;
for(int i = 0; i < nums.length; i++) {
    if(nums[i] != 0) {
        nums[writePos] = nums[i];
        writePos++;
    }
}
// Fill remaining positions with zeros
for(int i = writePos; i < nums.length; i++) {
    nums[i] = 0;
}
```
- Time: O(n), Space: O(1)
- Truly in-place solution
- Two passes but no extra space

**Approach 2: Swap Method (O(1) space) - MOST OPTIMAL**
```java
// Swap non-zero elements with the position at writePos
int writePos = 0;
for(int i = 0; i < nums.length; i++) {
    if(nums[i] != 0) {
        // Swap nums[i] with nums[writePos]
        int temp = nums[writePos];
        nums[writePos] = nums[i];
        nums[i] = temp;
        writePos++;
    }
}
```
- Time: O(n), Space: O(1)
- Single pass solution
- Minimizes total operations (follow-up requirement)
- Most efficient approach
