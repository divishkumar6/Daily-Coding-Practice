### 1. Move Zeroes

- **Difficulty**: Easy
- **Topic**: Arrays
- **Source**: LeetCode #283
- **Solution**: `Week-1/Arrays/MoveZeroes.java`
- **Approach**: Use temporary array to collect non-zero elements, then copy back
- **Time Complexity**: O(n) - two passes through the array
- **Space Complexity**: O(n) - temporary array of same size

**Notes:**
- First pass: Copy all non-zero elements to temp array in order
- Second pass: Copy temp array back to original (zeros automatically at end)
- Maintains relative order of non-zero elements
- Simple and straightforward approach
- Works correctly but uses extra space

**Current Solution Analysis:**
- ✅ Works correctly and passes all test cases
- ✅ Maintains relative order of non-zero elements
- ✅ Easy to understand and implement
- ⚠️ Uses O(n) extra space for temporary array
- ⚠️ Not truly in-place as problem suggests

**Future Optimization Ideas:**

**Approach 1: Two-Pointer Technique (O(1) space) - OPTIMAL**
```java
// Use two pointers: one for reading, one for writing non-zeros
int writePos = 0;
for(int i = 0; i < nums.length; i++) {
    if(nums[i] != 0) {
        nums[writePos] = nums[i];
        writePos++;
    }
}
// Fill remaining positions with zeros
for(int i = writePos; i < nums.length; i++) {
    nums[i] = 0;
}
```
- Time: O(n), Space: O(1)
- Truly in-place solution
- Two passes but no extra space

**Approach 2: Swap Method (O(1) space) - MOST OPTIMAL**
```java
// Swap non-zero elements with the position at writePos
int writePos = 0;
for(int i = 0; i < nums.length; i++) {
    if(nums[i] != 0) {
        // Swap nums[i] with nums[writePos]
        int temp = nums[writePos];
        nums[writePos] = nums[i];
        nums[i] = temp;
        writePos++;
    }
}
```
- Time: O(n), Space: O(1)
- Single pass solution
- Minimizes total operations (follow-up requirement)
- Most efficient approach

### 2. Linear Search

- **Difficulty**: Easy
- **Topic**: Arrays, Searching
- **Source**: TUF+ (Take U Forward)
- **Solution**: `Week-1/Arrays/LinearSearch.java`
- **Approach**: Sequential search through array from start to end
- **Time Complexity**: O(n) - worst case checks all elements
- **Space Complexity**: O(1) - no extra space needed

**Notes:**
- Iterate through array sequentially
- Return index immediately when target is found (first occurrence)
- Return -1 if target not found after checking all elements
- Simple and fundamental searching algorithm
- Works on both sorted and unsorted arrays

**Algorithm Analysis:**
- ✅ Optimal for unsorted arrays
- ✅ Simple to implement and understand
- ✅ O(1) space complexity
- ✅ Finds first occurrence automatically
- ⚠️ O(n) time complexity - not efficient for large sorted arrays

**When to Use Linear Search:**
- Unsorted arrays (no better option)
- Small arrays (overhead of other algorithms not worth it)
- When you need to find first occurrence
- When array is accessed sequentially anyway

**Alternative for Sorted Arrays:**
- Binary Search: O(log n) time complexity
- Much more efficient for large sorted datasets
