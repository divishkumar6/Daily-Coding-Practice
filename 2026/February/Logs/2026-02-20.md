### 1. Remove Duplicates from Sorted Array

- **Difficulty**: Easy
- **Topic**: Arrays
- **Source**: LeetCode #26
- **Solution**: `Week-1/Arrays/RemoveDuplicatesFromSortedArray.java`
- **Approach**: Count unique elements, create temp array, copy back to original
- **Time Complexity**: O(n) - three passes through the array
- **Space Complexity**: O(k) - temporary array for k unique elements

**Notes:**
- First pass: Count unique elements to determine k
- Second pass: Store unique elements in temporary array
- Third pass: Copy unique elements back to original array
- Works correctly for in-place modification requirement
- Handles duplicates by comparing with previous element

**Current Solution Analysis:**
- ✅ Works correctly and passes all test cases
- ✅ Easy to understand and maintain
- ⚠️ Uses extra space O(k) for temporary array
- ⚠️ Makes three passes through the array

**Future Optimization Ideas:**
- Can be optimized to O(1) space using two-pointer technique
- Single pass solution possible by maintaining write pointer
- No need for temporary array - can modify in-place directly

### 2. Rotate Array

- **Difficulty**: Medium ⭐
- **Topic**: Arrays
- **Source**: LeetCode #189
- **Solution**: `Week-1/Arrays/RotateArray.java`
- **Approach**: Use temporary array with modulo arithmetic for rotation
- **Time Complexity**: O(n) - two passes through the array
- **Space Complexity**: O(n) - temporary array of same size

**Notes:**
- Used modulo operator `(i + k) % nums.length` to handle wrap-around
- Handles edge case where k > array length automatically
- First pass: Calculate new positions and store in temp array
- Second pass: Copy back to original array
- Clean and straightforward solution

**Current Solution Analysis:**
- ✅ Works correctly for all test cases
- ✅ Handles k > array length edge case
- ✅ Simple to understand and implement
- ⚠️ Uses O(n) extra space for temporary array
- ⚠️ Not optimal for space complexity

**Future Optimization Ideas:**

**Approach 1: Reverse Method (O(1) space)**
```
1. Reverse entire array
2. Reverse first k elements
3. Reverse remaining n-k elements
Example: [1,2,3,4,5,6,7], k=3
Step 1: [7,6,5,4,3,2,1]
Step 2: [5,6,7,4,3,2,1]
Step 3: [5,6,7,1,2,3,4]
```

**Approach 2: Cyclic Replacements (O(1) space)**
- Use cyclic swaps to place elements directly in final positions
- More complex but achieves O(1) space
